## Rules

1. 전체적인 설명은 md파일로 docs에 정리합니다.
2. NEXT.md는 다음 계획과 요구사항을 기록해놓는 파일입니다. 현재 적힌 것처럼 이름, 날짜를 명시해주시기 바랍니다.
3. github를 씁니다. commit 할 때 반드시 커밋내용을 남겨주시기 바랍니다.
4. 브랜치는 main이 통합 브랜치입니다. 각자 하실 때는 git pull origin main 하셔서, 독립된 브랜치(feature/*)에서 개발하시고 push 하면 됩니다.
예를 들어, 로그인 기능을 개발하고 싶다 
->
git pull origin main
git checkout -b feature/login
여기서 개발 후,
git push origin feature/login

# 폴더별 역할 정리 (FOLDER_ROLES)

이 문서는 프로젝트 폴더별 책임을 한눈에 파악할 수 있도록 정리한 가이드입니다.

## 전체 구조

```text
project-root/
├─ backend/
│  ├─ config/
│  ├─ controllers/
│  ├─ mappers/
│  ├─ models/
│  └─ services/
├─ database/
├─ docs/
└─ frontend/
   ├─ css/
   ├─ js/
   └─ pages/
```

## backend

- 백엔드 API 서버 전체를 담는 최상위 폴더입니다.
- 프론트엔드와의 통신, 비즈니스 규칙 처리, 데이터 접근을 계층별로 분리해 관리합니다.

### backend/config
- 서버 실행/연결에 필요한 설정값을 관리합니다.
- 서버 포트, DB 연결, CORS/환경 설정 등 앱 구동에 필요한 공통 설정을 모아 둡니다.

### backend/controllers
- 라우트 요청을 받는 진입점 계층입니다.
- 요청 바디/파라미터를 읽고, 서비스 계층을 호출해 응답을 반환합니다.
- “누가 어떤 API를 호출할 때 무엇을 반환하는지”를 규정합니다.

### backend/services
- 실제 비즈니스 로직을 처리합니다.
- 유효성 검사, 정책 처리, 도메인 규칙을 수행하고, 저장소 계층에 작업을 위임합니다.

### backend/mappers
- 데이터베이스 접근 계층입니다.
- SQL 실행, 조회/삽입/수정/삭제 처리 등 영속화 작업을 담당합니다.
- 서비스 계층과 DB 사이의 변환/쿼리 책임을 수행합니다.

### backend/models
- 데이터 형태(도메인 엔티티/모델) 정의를 담당합니다.
- 사용자, 재료, 레시피, 게시글, 요청 등 객체 구조와 타입 기반 설계를 관리합니다.

## frontend

- 사용자 화면(HTML/CSS/JS)을 담당하는 클라이언트 영역입니다.
- 화면별 페이지, 스타일, 클라이언트 로직을 분리해 관리합니다.

### frontend/pages
- 실제 브라우저에서 보이는 페이지 파일을 모아 둡니다.
- 메인/로그인, 재료, 레시피, 커뮤니티, MBTI, 프로필 등 사용자 화면을 제공합니다.

### frontend/css
- 전역 스타일 시트를 담당합니다.
- 레이아웃, 컴포넌트 스타일, 반응형 UI 정의를 관리합니다.

### frontend/js
- 브라우저 동작 로직을 담당합니다.
- 공통 엔트리(`app.js`)와 기능 모듈(`modules`)로 나뉩니다.

#### frontend/js/modules
- 인증, 저장소, UI, 재료, 레시피, 커뮤니티, 레벨/경험치, MBTI 등 기능별로 분리된 모듈 폴더입니다.
- `app.js`에서 필요한 모듈을 조합해 화면별 동작을 연결합니다.

## database

- DB 스키마 및 초기 데이터 관련 파일을 관리합니다.
- 프로젝트에서 실제 사용할 테이블 구조와 초기값 정의를 둡니다.

## docs

- 프로젝트 문서 보관소입니다.
- 폴더 구조, 규칙, 실행/개발 안내 등의 운영 문서를 정리합니다.

## 정리

- `backend`는 요청 처리부터 DB 반영까지의 서버 책임을,  
- `frontend`는 페이지와 사용자 인터랙션을,  
- `database`는 데이터 저장 구조를,  
- `docs`는 프로젝트 규칙/설계를  
각기 분리해 관리하는 구조입니다.
